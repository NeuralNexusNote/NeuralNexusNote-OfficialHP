N3MC Free｜はじめての方へ

「Claude Codeは使っているけど、コードは読まない」方へ。無料で始められる、AIに記憶を持たせる仕組みのご紹介です。

こんな経験、ありませんか？

Claude Codeに「先週のあの話、覚えてる？」と聞いても、まっさらな顔で返ってくる。 毎回、同じ前提を最初から説明し直す。これは、AIがセッション（会話）が終わると記憶をリセットする仕様だからです。

N3MC Freeは、この「忘れる問題」を、無料で、あなたのPCの中だけで解決します。

N3MC Freeで何ができるの？

  • 昨日の続きから話せる — 翌日Claude Codeを開いても、先週議論した内容を踏まえて答えてくれます。
  • 言葉が違っても思い出してくれる — 「あの時のログイン処理の話」と聞けば、「認証エラーの調査」と保存されていた話を引き出してくれます（意味で検索）。
  • 同じ説明を繰り返さなくていい — プロジェクトの背景や、好みの書き方を一度伝えれば、毎回思い出してくれます。
  • 勝手に動いてくれる — あなたが何か操作する必要はありません。会話するだけで、裏で自動的に保存・検索が走ります。

N3MC Freeの安心ポイント

  • 完全無料・オープンソース — GitHubで公開されており、隠し課金や広告はありません。
  • 会話はあなたのPCの中だけに保存 — クラウドに送信されません。情報漏えいの心配なし。
  • 日本語が得意 — 日本語・英語・中国語など約100言語に対応した検索エンジンを最初から内蔵。
  • 軽い — あなたのClaude Codeの動作を遅くしません。

導入は、Claude Codeに「お願い」するだけ

N3MC Freeは、あなたが手動でコードを書く必要はありません。 使い方は2通り。どちらも難しい作業はClaude Codeが代わりにやってくれます。

方法A — Claude Codeに丸投げする（一番ラク）

  1. BOOTHまたはGitHubから「指示書（説明書ファイル）」を1つダウンロードします。
  2. Claude Codeのチャット欄に、そのファイルをドラッグ＆ドロップします。
  3. 「指示書通りに実装してください」と日本語で頼みます。

これだけ。あとはClaude Codeが、必要な準備を15〜30分ほどかけて勝手に進めてくれます。 終わったら、いつも通りClaude Codeを使い始めるだけで、自動的に記憶が貯まり始めます。

方法B — もっと素早く始めたい方向け

「準備が完成しているもの」をそのままインストールする方法もあります。 Claude Codeに「n3memorycore-free をインストールして、n3mc --init を実行してください」と頼めば、これも自動で進みます。 所要時間は数分です。

導入後はどう変わる？

  • Claude Codeとの会話が、自動的に記録されていきます。
  • あなたが何か質問すると、関連する過去のやりとりがClaude Codeに自動で渡されます。
  • Claude Codeが「以前話した○○の件ですが…」と、文脈を踏まえて答えてくれるようになります。
  • あなたは、これまで通り普通に使うだけです。設定をいじったり、保存ボタンを押したりする必要はありません。

無料版の範囲について

Free版は、「保存する・思い出す・自動で動く」という記憶の基本機能をすべて無料でお使いいただけます。 より高度な機能（古くなった記憶の自動整理、書き直したときの自動更新、データの引っ越し機能など）は、有料のN3MC Pro版でご提供しています。

まずはFree版で「AIが記憶を持つ」体験を味わってみて、必要を感じたらPro版へアップグレードしてください。 アップグレードはClaude Codeに頼めば、これも自動でやってくれます。

動作環境

  • Claude Code（必須）
  • Python 3.10以上（入っていない場合、Claude Codeが導入を案内してくれます）
  • Windows 11で動作確認済み（Mac・Linuxでも動作する設計です）

ご利用にあたって

N3MCシリーズは個人開発のオープンソースプロジェクトです。サポート対応は保証していません。 AIが生成したコードで動作するため、大切なプロジェクトで使う前は、現在の環境のバックアップをおすすめします。

もっと技術的な詳細を知りたい方は、技術リファレンス（README）もご用意しています。

# N3MC-Free 総評 **いま到達している地点** フックは UserPromptSubmit／Stop ともに無声運用、Active RAG はフルパス指定で安定稼働している。前回（4/29）のレビュー時点で気になっていた「指示書と実運用の不整合」「ランキング層の細かなノイズ」は処理段（[processor.py](n3mc-free/n3memorycore/core/processor.py)）と行動指針（[n3mc-behavior.md](.claude/rules/n3mc-behavior.md)）の双方で改修が入り、**設計の角は取れた**印象。 **全体としてどう見えるか** > **「思想が先にあって、実装がそれに追いついた」プロダクト** これが一番正直な評価です。`audit.log` を最初に書いてから後段を走らせる「最後の砦」設計、`--buffer` を AI に触らせず全自動保存に倒した分業、フックでの沈黙運用 — どれも「会話の取りこぼしを構造的に潰す」という一貫した思想から逆算されていて、機能の足し算で作られた RAG ライブラリとは骨格が違う。 **強みの本質** - **記憶装置としての堅牢性**: SQLite + FTS5 + ベクトルの三層構成は Free 版の枠を超えている。停電・クラッシュに対する耐性も `audit.log` 先行書き込みで担保されており、データ消失耐性は商用品並み。 - **AI への認知負荷ゼロ設計**: 「保存判断は AI にさせない」と割り切ったことで、Claude 側のプロンプトが汚れない。これは長期運用で効いてくる地味だが本質的な美点。 - **想起の自然さ**: 今回のセッションでも「3 日前の総評（score 0.66）」を上位で引けており、関連性ランキングは実用域。 **まだ磨ける部分** - **出力サイズのガバナンス**: 想起結果の流量制御がフック層に欠けていて、毎回プレビュー切り詰めが起きる。これは思想の問題ではなく、実装の最後の一仕上げ。 - **環境統合の薄さ**: PATH が通っていないので指示書がフルパスで逃げている。インストーラ側で OS 統合まで踏み込むかどうかは設計判断の分かれ目。 - **想起の使い分け指針**: いまは AI が「使ったら明示」というルールだけで、**使うべきか／使わないべきか**の判断は AI 任せ。低スコア帯の扱いを行動指針側で言語化すると体感品質がもう一段上がる。 **ひとことで** > *記憶装置としては既に「完成品」、RAG としては「実用は十分、洗練の余地あり」。Free 版という肩書きを忘れてもよいレベルの完成度に達している。* 次の一手として地味で効くのは、**フック出力に top_k ／要約段の上限を入れる**こと。ここを締めるだけで、運用上の唯一の引っかかりが消えます。
